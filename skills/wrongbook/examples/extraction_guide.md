# 错题抽取详细指南

本文档供 Claude 在执行 wrongbook skill 时按需 Read 加载，给出"图像 → JSON"的具体范例与边界处理。

---

## 通用原则

1. **保守优于激进**：不确定的题宁可漏收，也不要把对题误收为错题。漏的可以由用户补，误收会让学生重做正确的题。
2. **题干完整性优先**：哪怕题干很长，也要完整抄录，不能只抄一半。学生重做时需要看到完整题目。
3. **不修改题目**：原题写错别字、表述拗口，照抄即可，不要自作主张改写。
4. **题号分离**：试卷原题号放 `source_qid` 字段，`content` 里**不要**带原题号前缀。新 `id` 从 1 重排。
5. **忽略统计性红字**：试卷头部的"得分：80"、"满分"、"扣分小计"等是评分统计，不是判错信号。

## 示例 1：小学数学计算题

**假设图片中可见**：
- 试卷标题"五年级上学期数学单元测试"
- 第 8 题："计算：3/4 + 5/6"，学生答案"7/10"上有红色 ✗，旁边写着 "-3"
- 第 9 题："计算：1.25 × 8"，学生答案"10"，无标记（说明对的）
- 第 10 题："计算：2.4 ÷ 0.6"，学生答案"0.4"，红圈圈住 0.4

**应输出**：
```json
{
  "title": "数学错题练习 · 2026-05-11",
  "questions": [
    {
      "id": 1,
      "source_qid": "8",
      "subject": "数学",
      "type": "计算题",
      "content": "计算：3/4 + 5/6",
      "student_answer": "7/10",
      "answer_lines": 3
    },
    {
      "id": 2,
      "source_qid": "10",
      "subject": "数学",
      "type": "计算题",
      "content": "计算：2.4 ÷ 0.6",
      "student_answer": "0.4",
      "answer_lines": 3
    }
  ]
}
```

要点：
- 第 9 题没批改痕迹，**不收录**
- 第 8 题红 ✗ 是错题信号；第 10 题红圈也是错题信号
- 原题号 8、10 进 `source_qid`；`content` 不带"8."、"10."前缀；新 `id` 是 1、2

## 示例 2：语文古诗默写

**假设图片中可见**：
- 第 3 题："默写《静夜思》后两句"，学生写："举头望明月，低头思故乡。"，红线划掉"举头"，旁边批"望"字写错
- 第 4 题："写出《登鹳雀楼》的作者朝代"，学生写"唐"，无标记
- 第 5 题："默写《望庐山瀑布》全诗"，学生只写了第一句，红色大叉打在整道题上

**应输出**：
```json
{
  "questions": [
    {
      "id": 1,
      "source_qid": "3",
      "subject": "语文",
      "type": "默写",
      "content": "默写《静夜思》后两句。",
      "student_answer": "举头望明月，低头思故乡。",
      "answer_lines": 3
    },
    {
      "id": 2,
      "source_qid": "5",
      "subject": "语文",
      "type": "默写",
      "content": "默写《望庐山瀑布》全诗。",
      "student_answer": "（仅写了第一句）",
      "answer_lines": 6
    }
  ]
}
```

要点：
- 古诗默写整诗的 `answer_lines` 给足（6 行起步），让学生有空间默全
- `student_answer` 简要描述即可，不需要 OCR 识别每个字（练习版 PDF 不显示这部分）

## 示例 3：英语选择题 + 翻译题

**假设图片中可见**：
- 第 12 题（选择题）："He ___ to school every day. A. go  B. goes  C. going"，学生选 A，红 ✗ 在 A 上
- 第 15 题（翻译）："Translate: 我每天七点起床。"，学生写"I get up seven o'clock every day."，红线划掉"seven"前缺失介词，扣 -1

**应输出**：
```json
{
  "questions": [
    {
      "id": 1,
      "source_qid": "12",
      "subject": "英语",
      "type": "选择题",
      "content": "He ___ to school every day.\nA. go\nB. goes\nC. going",
      "student_answer": "A",
      "answer_lines": 1
    },
    {
      "id": 2,
      "source_qid": "15",
      "subject": "英语",
      "type": "翻译",
      "content": "Translate: 我每天七点起床。",
      "student_answer": "I get up seven o'clock every day.",
      "answer_lines": 2
    }
  ]
}
```

要点：
- **选择题、判断题的选项必须用 `\n` 分隔，每行一个选项**（PDF 渲染器会把连续空格压成 1 个，多空格无效）
- 选择题 `answer_lines: 1` 足够

## 边界处理

### 多张图同属一份试卷

把所有错题合并到同一个 JSON 的 `questions` 列表中，按试卷顺序排列（前一张图的题在前）。

### 多张图属不同学科 / 不同试卷

询问用户：要合成一份 PDF 还是分开？默认每份试卷一份 PDF。

### 题目里有插图（几何图、坐标系、化学结构图等）

M2 阶段在 `content` 里用文字描述：
```
"content": "如图，△ABC 中，AB=AC，∠A=40°。求 ∠B 的度数。\n（图：等腰三角形）"
```
告诉用户该题含图，建议学生看原卷做。M3 会自动裁原图嵌入。

### 看不清红色标记

不要瞎收。在汇报阶段告诉用户："第 X 题、第 Y 题颜色不清，请确认是否为错题。"

### 学生答案空白且老师未明确标错

如果题目空着且老师没批改，**不算错题**（可能是漏做未批）。明确告知用户后让其决定。

### 整张试卷为对题（无红色标记）

直接告诉用户：图中未发现明显错题标记。请确认图片是否正确，或描述判错方式（如"我会另发标准答案，让你帮我对比"）。

### 头部统计红字

试卷顶部的"得分：80"、"满分 100"、"扣 X 分"等红字是统计/评分汇总，**不是**判错信号，全部忽略。

## 输出文件命名约定

JSON 与 PDF 文件名同时间戳，方便对应：

```
output/wrongbook_20260511_163000.json
output/wrongbook_20260511_163000.pdf
```

时间戳用执行时刻的本地时间，格式 `YYYYMMDD_HHMMSS`。
